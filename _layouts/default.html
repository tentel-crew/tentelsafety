TentelSafety Documentation

Welcome!

TentelSafety provides a security feature for the Discord server. The bot is built on algorithms and functions that initiate an addition to the secure environment on the server.

DoS Protect

TentelSafety identifies incoming integrations to the server. Unverified integrations will not be allowed to reach the server.

User Protect

TentelSafety identifies alternative accounts by comparing the standard flow conditions with the incoming user.

Status Check

The algorithm excludes incoming users with offline status from the server.

Permissions Protect

The algorithm analyzes the initial rights of the integration or user, and if they exceed the norm, TentelSafety initiates neutralization.

Set Quarantine

Database of violators of Discord community rules in the Russian segment of the Discord platform. TentelSafety bans incoming users that are identified in the database.

Verification

Flexible verification system equipped with heuristic analysis. TentelSafety analyzes the user undergoing verification, comparing it with the passage conditions specified by heuristic analysis.

Find out how to use: t.help > Functions

Bypass TentelSafety.

Currently, there are no malicious integrations capable of bypassing the DoS Protect TentelSafety. But there is a technical tool that is able to bypass the identification of the attack; the bot does not respond to self bots in the attacker's accounts. To counteract this, it is enough for you not to grant administrator rights, server management, webhook management, role management and channel management to questionable persons. (details of information security in Discord can be found on the Tentel Crew server)
